Git Kurulumu

1. Git Kurulumu ve Global Kullanıcı Bilgileri Ayarı

  • Termux'u açın.
  • pkg update
    - Paketleri güncellemek için kullanılır.
  • pkg install git
    - Git'i kurmak için kullanılır.

    • Global kullanıcı bilgilerini ayarlamak için

  • git config --global user.name "YourName"
    - Git kullanıcı adınızı ayarlar.
  • git config --global user.email "your_email@example.com"
    - Git e-posta adresinizi ayarlar.

    • SSH Anahtarı Oluşturma ve Düzenleme

  • SSH anahtarı oluşturmak için:
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
    SSH anahtarı oluşturur.
  • SSH anahtarını düzenlemek için:
    nano ~/.ssh/id_rsa.pub
    - SSH anahtarını düzenlemek için metin düzenleyiciyi açar.

GitHub'da Depo Oluşturma

  • Giriş Yapın: GitHub hesabınıza giriş yapın.
  • + (Artı) İşaretine Tıklayın: Sağ üst köşede bulunan "+ (Artı)" işaretine tıklayın ve "New repository" (Yeni depo) seçeneğini seçin.
  • Depo Adını ve bsd-vurguluyı Girin: Yeni depo oluşturma sayfasında, depo adını ve gerekirse bsd-vurguluyı girin.
  • Create Repository'ye Tıklayın: Bilgileri doldurduktan sonra "Create repository" (Depo oluştur) butonuna tıklayın.

SSH Anahtarlarını GitHub'a Eklemek

  • Giriş Yapın ve Ayarlar'a Girin: GitHub hesabınıza giriş yapın ve sağ üst köşede bulunan profil fotoğrafınıza veya hesap adınıza tıklayarak açılır menüyü açın. Menüden "Settings" (Ayarlar) seçeneğine tıklayın.
  • SSH ve GPG Keys'e Gidin: Sol menüden "SSH and GPG keys" (SSH ve GPG anahtarları) seçeneğine gidin.
  • Yeni SSH Anahtarı Ekleyin: "New SSH key" (Yeni SSH anahtarı) veya benzer bir butona tıklayarak yeni SSH anahtarınızı ekleyin.
  • Anahtarı Kopyalayın ve Kaydedin: SSH anahtarınızı oluşturduktan sonra, SSH anahtarınızın adını ve anahtarın kendisini belirten metni kopyalayın ve kaydedin.

Depoyu Git ile Bağlama

  • Depo Klasörü Oluşturun: Projeyi saklayacağınız bir klasör oluşturun ve içine girin.
    mkdir my_project
    cd my_project
  • Git'i Başlatın:
    Klasörü bir Git deposu olarak başlatın:
    git init
  • Depo Durumunu Kontrol Edin: Hangi dosyaların izlendiğini ve hangilerinin izlenmediğini kontrol edin:
    git status
  • Geçmiş Commit'leri Görüntüleyin:
    git log
